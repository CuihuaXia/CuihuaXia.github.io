win10安装Git

①从 Git 官网 https://git-scm.com/downloads 直接下载安装程序；

②按默认选项安装，即可；

③安装完成后，会出现三个终端入口：Git GUI，Git Bash，Git CMD。在开始菜单里找到 “Git”->“Git Bash”，蹦出一个类似命令行窗口的东西，就说明 Git 安装成功；

④安装完成后，还需要最后一步设置，在命令行输入：

$ git config --global user.name “Your Name"
$ git config --global user.email “Your Email Address"

表示你这台机器上所有的 Git 仓库都会使用这个配置。

小贴士：如果用户名写错了，可以重置账号：

$ git config --global --replace-all user.name "输入你的用户名"
$ git config --global --replace-all user.email "输入你的邮箱"

⑤使用Git从GitHub网站克隆仓库到本地：git clone 仓库地址

cd D:/GitHubDesktop/test
git clone https://github.com/cuihuaxia/cuihuaxia.github.io

出现错误一

Failed to connect to github.com port 443: Timed out

原因

可能是此win10电脑无法连接github.com网站。

解决办法

尝试：

ping github.com 不成功，ping www.baidu.com 成功，说明可以正常上网；

因此，需要确认github.com网址的相关IP：

第一步：打开 ipaddress.com 网站，查询如下两个域名，并分别记录下其对应的ip：① github.com ② github.global.ssl.fastly.net

查询结果如下：

140.82.112.3 github.com

199.232.69.194 github.global.ssl.fastly.net

第二步：更新host文件并保存。在host文件（C:\WINDOWS\System32\drivers\etc\hosts）中加入以下两行：

140.82.112.3 github.com
199.232.69.194 github.global.ssl.fastly.net

小贴士

Win10系统hosts文件在哪：在Windows系统中，hosts文件都是位于系统盘的Windows文件夹夹下，文件位置路径如下：C:\WINDOWS\system32\drivers\etc；

Win10快速打开hosts文件位置方法：首先使用 Win + R 组合快捷键打开运行命令框，然后在打开后面键入或者粘贴上面的hosts文件路径，点击“确定”就可以快速打开hosts文件位置了；

如果etc文件夹中并没有hosts文件，那么可能是该文件隐藏了，设置一下，让其显示出来即可。

取消/重置Git全局代理：

git config --global --unset http.proxy
git config --global --unset https.proxy

出现错误二

Permission denied

原因

需要创建ssh key并添加ssh key到github。

解决办法

①检查本地ssh key是否存在：

打开git bash，输入以下命令检查ssh key是否存在

ls ~/.ssh/

若存在，跳过步骤②，不存在则执行步骤②；

②生成ssh key：

在git bash中执行以下命令：

ssh-keygen -t rsa -b 2048 -C "自己的邮箱地址"

若出现　

Generating public/private rsa key pair.
Enter file in which to save the key (/c/Users/***/.ssh/id_rsa):

***代表当前登录用户名，不做修改直接回车，会将生成的rsa文件保存为默认名称；

回车之后提示设置提交/拉取代码到GitHub时需要的密码及确认密码：

Enter passphrase (empty for no passphrase):
Enter same passphrase again:

设置密码后再次回车提示Your identification has been saved in.... 即表示ssh key生成成功；

③添加ssh key至ssh-agent：

执行以下命令

eval `ssh-agent`

如果返回以下表示启用中

执行以下命令添加ssh key至ssh-agent

ssh-add ~/.ssh/id_rsa

此步会要求输入步骤②设置的密码，输入密码后会出现：

此步若是报错：

可以尝试关掉当前git bash窗口，重新以管理员身份运行git bash

④添加ssh key至github：

复制/c/Users/Dell18/.ssh/id_rsa.pub中的全部内容（以ssh-rsa开头）；

登录 https://github.com/ ，在页面右上角自己头像右边箭头处右击，弹框中进入Setting功能；

Setting界面右边菜单选择SSH and GPG keys，选择新建SSH keys；

粘贴刚才复制的内容，然后保存即可。

完成以上步骤就可以使用ssh方式，用命令git clone [git@github.com](mailto:git@github.com):***.git的形式下载项目到本地了。

win10安装GitHub Desktop

①从官网 https://desktop.github.com/ 直接下载安装程序GitHub Desktop；

②按默认选项安装，即可；

③安装完成后，即打开终端页面；

④在github.com中复制一个仓库地址 https://github.com/CuihuaXia/Statistics ，然后克隆此仓库到本地，即可：

出现错误一

Windows下安装的GitHub Desktop在选择File-Options-Accounts更改账户时，账户页面闪退并弹出对话框“Untrusted server”错误

原因

由于中途更换电脑和git账户信息，导致项目不能clone，后来发现是本地win10系统保存原来的git账户信息所致，这时修改git账户信息后即可。

解决办法

删除或修改本地Git或GitHub保存的账号密码：

Win+R+control 快捷键打开控制面板——用户账户——凭据管理器——管理Windows凭据——删除和git或GitHub相关的普通凭据。

此时再进入GitHub Desktop更改账户时，会正常显示账户页面。

出现错误二

在GitHub Desktop登录账户时，出现错误“failed to fetch”

原因

网络错误：防火墙或DNS问题。 在中国很难访问github，连接GitHub网站的网络很不稳定

解决办法

办法①：在终端运行VPN

办法②：不使用账号密码登录，使用从浏览器登录方式，然后多试几次，总有一刻可以连的上。

小贴士

①GitHub官网已经给出使用GitHub Desktop常见的可能会出现的问题及相应的解决办法：https://docs.github.com/cn/free-pro-team@latest/desktop/installing-and-configuring-github-desktop/authenticating-to-github

②Win10 GitHub Desktop的安装，进入官网 https://desktop.github.com/ 后，直接下载.exe应用程序，.msi程序不是安装程序。
