前言

对于不喜欢做重复工作又时常没有安全感的小编来说，数据安全很重要，因此博客源文件备份是一个极其重要的事情。

那如何简单、快捷又及时地将博客源文件做好备份呢？

可能有的小伙伴说，准备一个U盘或移动硬盘，或者用云盘或云笔记（比如百度网盘、印象笔记）定期（比如每两周）做一个备份，不就好了？

是的，我最初就是用这个方法做博客备份的。然鹅～记性不好又怕麻烦的我，觉得这个方法仍然不是很保险，也不怎么高效和及时。我希望找到一个既能保证我的博客源文件安全地做好备份，又能够及时同步备份我更新的博客内容的办法。在网上一搜，果然，这样的方法还真有！

有童鞋问了，博客网站不是托管在GitHub Pages上吗，那博客源文件不是会直接同步在github.io仓库里吗？但，你仔细去看下github.io仓库中的文件就会发现，Hexo博客是静态托管的，github.io仓库只有生成的静态网页文件，并没有Hexo的源文件。

咦，是不是像当初的我一样，傻眼了？

另外，博客源文件备份还有两个原因：①如果电脑突然罢工，我们的源文件不会丢失；②有时候不方便需要更换电脑写作，我们不必携带任何外在物件比如U盘进行拷贝等繁琐操作，只需直接clone仓库就可以了。

明白了为什么要进行博客源文件备份之后，下面马上进入正题。

因为我之前已经搭建博客一段时间了，直接进行博客源文件在GitHub上的备份，行不通，因此我直接从最初的Hexo搭建博客步骤开始。因为我的博客是在Mac上搭建的，所以以下是展示的在Mac上使用Hexo搭建博客以及使用GitHub进行博客源文件备份的步骤，Win下的操作，大致类似，这里不进行具体描述。

在本地Docments下新建一个文件夹MyBlog

右键打开Terminal/iTerm2 (gitbash here)

超级用户下安装hexo

sudo npm install -g hexo

进入MyBlog目录

cd /Users/xia/Documents/MyBlog

hexo初始化

hexo init

注意：hexo init 会初始化 Hexo 环境，该操作会清空.git 文件夹, 导致版本控制信息会丢失。因此除了首次搭建环境之外, 无需运行此命令。

使用npm安装所需依赖

npm install
npm install hexo-deployer-git

修改根目录下config文件中为git, .github.io.git, master

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repository: https://github.com/cuihuaxia/cuihuaxia.github.io.git
  branch: master

拷贝自己的文件到MyBlog目录下

  1. 文件夹（文件夹里的内容是合并和替代）：
    public, scaffolds, scripts, source, themes.
  2. 文件（替代）：
    _config.yml, db.json, package-lock.json, package.json, README.md.

初始化git本地版本管理仓库

git init

在本地新建一个分支“backup”用于备份博客

git branch backup

切换到新建分支

git checkout backup

重新添加远程仓库地址

git remote add origin git@github.com:cuihuaxia/cuihuaxia.github.io.git

推送博客源文件到GitHub的backup分支

git add ./
git commit -m '备份博客'
git push origin backup

到这里,我们已经将我们的博客推送到GitHub的backup分支下面了。我们到Github的cuihuaxia.github.io仓库下会发现我们的项目多了一个backup分支，这个分支就是我们博客源文件的备份。

将Github的cuihuaxia.github.io仓库下的backup分支更改为默认分支

日常博文修改

# 将改动推送到GitHub (此时当前分支应为backup)
git checkout backup
git add ./
git commit -m '备份博客'
git push origin backup
# 发布网站到master分支上(此时当前分支应为master)
git checkout master
hexo cl && hexo g && hexo d

本地资料丢失/恢复博客

当重装电脑之后，或者想在其他电脑上修改博客，可以使用下列步骤 :

  1. 将 cuihuaxia.github.io 仓库 clone 到本地：

    # 拷贝仓库(默认分支为backup)
git clone [git@github.com](cuihuaxia:git@github.com):cuihuaxia/cuihuaxia.github.io.git
// git clone https://github.com/cuihuaxia/cuihuaxia.github.io

  2. 在本地重新安装 Hexo 编译环境。通过Git bash依次执行下列指令:

    npm install hexo
npm install
npm install hexo-deployer-git

注意：在此不需要执行hexo init这条指令，因为不是从零搭建新博客。
