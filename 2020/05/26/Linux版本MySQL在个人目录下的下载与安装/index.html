MySQL下载

官网下载地址：https://dev.mysql.com/downloads/

进入MySQL Community Downloads，选择MySQL Community Server；

进去之后是自己电脑操作系统对应的最新版本的MySQL，如下所示：

但是我并不想使用这个操作系统，也不想用最新版本的MySQL，那如果我想用Linux系统的MySQL 5.7，怎么办呢？

别慌，上图右侧不是有个Looking for previous GA version?吗？点击它，就会出现我想要的版本号5.7.30，然后选择操作系统为Source Code，选择OS Version为Generic Linux (Architecture Independent)，就会出现压缩包mysql-5.7.30.tar.gz；

点击下载，不必登录，不必注册，直接点击No thanks, just start my download即开始下载。

Linux (CentOS 7)版本MySQL 5.7.30在个人目录下的安装

下载安装boost_1_59_0.tar.gz

在安装MySQL 5.7.30之前，还需要先安装一下boost_1_59_0，这在cmake那一步会用到。

# 移除之前解压的boost_1_59_0文件夹及其文件
rm -R /zs32/home/chxia/Softwares/boost_1_59_0
# 下载boost_1_59_0.tar.gz
cd /zs32/home/chxia/Softwares
wget http://sourceforge.net/projects/boost/files/boost/1.59.0/boost_1_59_0.tar.gz
# 解压新的boost_1_59_0.tar.gz到安装路径
tar -zxvf boost_1_59_0.tar.gz

解压mysql-5.7.30.tar.gz

# 移除之前解压的mysql-5.7.30文件夹及其文件
rm -R /zs32/home/chxia/local/mysql-5.7.30
# 设置mysql-5.7.30的安装路径
cd /zs32/home/chxia/local
# 解压新的mysql-5.7.30.tar.gz到安装路径
tar -zxvf /zs32/home/chxia/Softwares/mysql-5.7.30.tar.gz

创建MySQL数据目录

cd /zs32/home/chxia
mkdir -p ~/data/mysql-5.7.30

MySQL的数据会存放在这里。

创建MySQL配置文件的目录

cd /zs32/home/chxia
mkdir -p ~/etc/mysql-5.7.30

这样，我们就不用去理会root权限才能操作的/etc/my.cnf了。

创建mysqld_safe的日志目录

如果你对mysqld_safe有印象，那你一定知道我们就是通过这个家伙来启动MySQL的。但是它默认打印日志在/var/log/mysqld.log。很显然这是ROOT才有的权限。我们普通用户不稀罕，自己提供日志目录，无拘无束。

cd /zs32/home/chxia
mkdir -p ~/logs/mysqld_safe/

cmake 构建MySQL

做好了上面的准备工作后，我们就可以开始安装MySQL了。

  1. 首先是用cmake构建出一个Makefile来。
cd /zs32/home/chxia/local/mysql-5.7.30
cmake . \
 -DCMAKE_INSTALL_PREFIX=$HOME/local/mysql-5.7.30 \
 -DMYSQL_DATADIR=$HOME/data/mysql-5.7.30 \
 -DSYSCONFDIR=$HOME/etc/mysql-5.7.30 \
 -DDOWNLOAD_BOOST=1 \
 -DWITH_BOOST=/zs32/home/chxia/Softwares/boost_1_59_0

很重要的一点就是通过-DCMAKE_INSTALL_PREFIX，-DMYSQL_DATADIR，-DSYSCONFDIR参数来设定自己的环境的：MySQL安装的目录，MySQL存放数据的目录，MySQL存放和读取配置的目录。

  1. 然后make和make install安装MySQL 5.7.30
make && make install

至此，MySQL已经成功安装到我们指定的目录下了。不信，您到$HOME/local/mysql-5.7.30目录下瞧瞧。

  1. 新建配置文件my.cnf到我们指定的配置目录下
touch ~/etc/mysql-5.7.30/my.cnf

修改配置文件配置MySQL

vi ~/etc/mysql-5.7.30/my.cnf
i



[client] 
password = 
port = 3506 
socket = /zs32/home/chxia/local/mysql-5.7.30/tmp/mysql.sock

[mysqld] 
port = 3506
socket = /zs32/home/chxia/local/mysql-5.7.30/tmp/mysql.sock 
skip-grant-tables 
skip-external-locking 
key_buffer_size = 384M 
max_allowed_packet = 1M 
table_open_cache = 512 
sort_buffer_size = 2M 
read_buffer_size = 2M 
read_rnd_buffer_size = 8M 
myisam_sort_buffer_size = 64M 
thread_cache_size = 8 
query_cache_size = 32M  
datadir = /zs32/home/chxia/local/mysql-5.7.30/data
language = /zs32/home/chxia/local/mysql-5.7.30/share/english

[mysql] 
auto-rehash
default-character-set=utf8
socket= /zs32/home/chxia/local/mysql-5.7.30/tmp/mysql.sock

[mysqld_safe] 
log-error=/zs32/home/chxia/logs/mysqld_safe/mysqld.log 
pid-file=/zs32/home/chxia/data/mysql-5.7.30/mysqld.pid



esc
:wq

参数解释：

  1. 默认端口port是3306，根据个人喜好修改。不过建议不要使用默认的端口，和同一台机器的同事冲突不说。关键是容易被别人攻击。
  2. datadir，顾名思义，就是MySQL存储具体数据的地方了。我们之前的准备工作就是为了它。
  3. auto-rehash。这个配置项默认是no-auto-rehash。我把它注释掉了，改成了auto-rehash。好处就是使用mysql客户端连接的时候，在命令行里面输入sql的时候，tab键可以起到自动补全的作用哦。非常方便和实用！
  4. mysqld_safe。这样我们使用mysqld_safe来启动mysql的时候，就可以顺利启动了。毕竟日志文件的目录是我们为他专门设定的。

初始化MySQL

接下来就可以初始化MySQL了，建立一些MySQL本身需要的库表等基础数据。

# 查看并杀掉MySQL已有进程
lsof |grep nfs00000
kill -9 27729
# 删除local/mysql-5.7.30/data中的数据
rm -R /zs32/home/chxia/local/mysql-5.7.30/data/
# MySQL初始化
/zs32/home/chxia/local/mysql-5.7.30/bin/mysqld --initialize --basedir=/zs32/home/chxia/local/mysql-5.7.30/ --datadir=/zs32/home/chxia/local/mysql-5.7.30/data/
# 删除已有mysql.sock文件
rm /zs32/home/chxia/local/mysql-5.7.30/tmp/mysql.sock

注意：下面这些代码暂时不用，没用。

## 创建系统数据库的表
# 很多网上的文章都会加上 –user=mysql。这也曾经让我很困惑，我是普通用户，和mysql这个用户有什么关系呢。今天终于明白，只有root用户才有资格使用这个参数来指定哪个用户运行mysqld，不指定的话，就是当前登陆用户。所以我们没必要带上这个参数了。no-defaults 就是完全不使用默认的东西，我们就是要用完全属于自己的MySQL。–defaults-file就是指定我们自己的配置文件。
cp /zs32/home/chxia/local/mysql-5.7.30/bin/mysql_install_db /zs32/home/chxia/local/mysql-5.7.30/scripts/mysql_install_db
/zs32/home/chxia/local/mysql-5.7.30/scripts/mysql_install_db –no-defaults –defaults-file=/zs32/home/chxia/etc/mysql-5.7.30/my.cnf --datadir=/zs32/home/chxia/local/mysql-5.7.30/data/data

# 修改密码
/zs32/home/chxia/local/mysql-5.7.30/bin/mysqladmin -u root password 'new-password'
/zs32/home/chxia/local/mysql-5.7.30/bin/mysqladmin -u root -h host4 password 'new-password'

大功告成，启动和登录MySQL

# 启动MySQL
/zs32/home/chxia/local/mysql-5.7.30/bin/mysqld_safe --defaults-file=/zs32/home/chxia/etc/mysql-5.7.30/my.cnf &
# 登录MySQL(直接在上一条命令基础上输入这一条命令即可)
/zs32/home/chxia/local/mysql-5.7.30/bin/mysql -u root -p -S /zs32/home/chxia/local/mysql-5.7.30/tmp/mysql.sock //无密码，直接回车即可
# 修改root登录密码并退出mysql
use mysql;
update mysql.user set authentication_string=password('new-password') where user='root';
flush privileges;
exit;

设置环境变量，使系统默认的mysql为自己新安装的mysql

# 设置环境变量，使默认软件/程序mysql为自己新安装的mysql
vi ~/.bashrc
i


export PATH=/zs32/home/chxia/local/mysql-5.7.30/bin:/zs32/home/chxia/local/mysql-5.7.30/lib:$PATH


esc
:wq
# 使.bashrc文件设置生效
source ~/.bashrc

之后便可以直接输入mysql进行登录了

至此，安装mysql的全过程结束，下次便可以直接登录mysql了。

# 登录mysql（输入密码，即可登录）
mysql -u root -p

最期待，最激动人心的时刻来了！

至此，Linux版本的MySQL 5.7.30在个人用户/个人账户（非root账户）下的安装圆满成功！撒花～～～
