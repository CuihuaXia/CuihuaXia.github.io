安装包下载

  1. 去官网 http://httpd.apache.org/下载Apache的源码包，这里下载的是最新的httpd-2.4.43.tar.bz2这个包。
  2. 去官网http://apr.apache.org/download.cgi下载apr和apr-util的源码包，这里下载的是最新的apr-1.7.0.tar.gz和apr-util-1.6.1.tar.gz这两个包。
  3. 去官网https://ftp.pcre.org/pub/pcre/下载pcre的源码包，这里下载的是pcre-8.44.zip这个包。

解压并安装apr

# 解压apr
cd /zs32/home/chxia/Softwares
tar -zxvf apr-1.7.0.tar.gz
# 进入源码包目录，安装apr
cd apr-1.7.0
./configure --prefix=/zs32/home/chxia/Softwares/apr-1.7.0/usr/local/apr
make
make install

上面的配置参数–prefix就是指定安装目录，执行没问题稍等一会就安装好了。

解压并安装apr-util

# 解压apr-util
cd /zs32/home/chxia/Softwares
tar -zxvf apr-util-1.6.1.tar.gz 
# 进入源码包目录，安装apr-util
cd apr-util-1.6.1
./configure --prefix=/zs32/home/chxia/Softwares/apr-util-1.6.1/usr/local/apr-util --with-apr=/zs32/home/chxia/Softwares/apr-1.7.0/usr/local/apr
make
make install

上面配置参数的–with-apr表示apr-util安装需要依赖apr，所以指定apr的安装目录，到这里apr-util也安装完成了。

解压并安装pcre

# 解压pcre
cd /zs32/home/chxia/Softwares
unzip pcre-8.44.zip 
# 进入源码包目录，安装pcre
cd pcre-8.44
./configure --prefix=/zs32/home/chxia/Softwares/pcre-8.44/usr/local/pcre
make
make install

解压并安装Apache

# 删除之前安装的Apache httpd-2.4.43
rm -R /zs32/home/chxia/Softwares/httpd-2.4.43
## 解压并安装Apache httpd-2.4.43
# 解压Apache httpd-2.4.43
cd /zs32/home/chxia/Softwares
tar -jxvf httpd-2.4.43.tar.bz2
# 进入源码包目录，安装Apache httpd-2.4.43
cd /zs32/home/chxia/Softwares/httpd-2.4.43
./configure --prefix=/zs32/home/chxia/Softwares/httpd-2.4.43/usr/apache24 --with-apr=/zs32/home/chxia/Softwares/apr-1.7.0/usr/local/apr --with-apr-util=/zs32/home/chxia/Softwares/apr-util-1.6.1/usr/local/apr-util --with-pcre=/zs32/home/chxia/Softwares/pcre-8.44/usr/local/pcre
make clean
make
make install

编译过程时间有点长，耐心等待一下，执行完毕，Apache就安装完成了。

注意：如果make中出现错误，必须执行make clean清除所有缓存文件后继续，如果是在make install过程中出现错误，必须执行make clean并且删除生成了的安装目录，然后重新执行make和make install。

启动Apache httpd服务之前

  1. 修改端口为1024以上端口

    注：非root用户使用的端口80将无权限，1~1024 端口只有root只能绑定。你如果不是用root启动apache，则需修改为1024以上的端口。
    所以，需要打开/zs32/home/chxia/Softwares/httpd-2.4.43/usr/apache24/conf/httpd.conf修改端口信息。

    #Listen 12.34.56.78:80
 Listen 8080      //修改端口为8080或8008

  2. 打开/zs32/home/chxia/Softwares/httpd-2.4.43/usr/apache24/conf/httpd.conf文件找到#ServerName www.example.com:80, 把这行代码前面的#去掉，保存退出，然后重新启动Apache。

  3. 需要注意服务器的默认根目录（即放index.html等网站文件的目录）是在/zs32/home/chxia/Softwares/httpd-2.4.43/usr/apache24/htdocs/。

启动Apache httpd服务

# 启动Apache服务
/zs32/home/chxia/Softwares/httpd-2.4.43/usr/apache24/bin/apachectl start
/zs32/home/chxia/Softwares/httpd-2.4.43/usr/apache24/bin/apachectl -f /zs32/home/chxia/Softwares/httpd-2.4.43/usr/apache24/conf/httpd.conf -k start

浏览器访问对应服务器的外网ip地址（加端口），看到It works! 即表示安装成功

停止Apache httpd服务

# 停止Apache服务
/zs32/home/chxia/Softwares/httpd-2.4.43/usr/apache24/bin/apachectl stop

结束语

在任务如此繁重，时间如此紧张的情况下，今天，我又花了整整一天时间安装Apache服务，以为终于可以使用自己实验室的Linux服务器搭建网站，并可以成功访问网页了，然而——现实是残酷的。在老板不给租云服务器，而自己实验室的服务器又没有root操作权限的前提下，我的数据库网站搭建技术学得再好，也只是白费，想要以一己之力实现搭建数据库网站和发数据库文章一条龙的梦，始终是——泡沫。

因此，也许很快…我不会再坚持“作为一个生信人必须要掌握开发数据库网站这项基本技能”了吧。

科研的时间如此宝贵，我不会把宝贵的时间和精力浪费在“一开始就知道不会有结果”的事情上。

既然此路不通，那我从此便关闭此路。

科研之路，条条大路通罗马，马上开启自己的下一条路便是了。下一条路，大可能比这一条路容易实现得多。
